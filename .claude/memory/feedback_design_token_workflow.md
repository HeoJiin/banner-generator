바이브 코딩에서 디자인 토큰과 시스템 컴포넌트 작업 순서.

## Phase 1: 기능 만들 때 — 최소 토큰만 깔고 시작

코딩 전 5분:
- 색상 5개 (primary, secondary, muted, border, destructive)
- 타이포 3개 (caption, body, heading)
- 스페이싱 3개 (좁음, 기본, 넓음)
- 라디우스 2개 (sm, md)

하드코딩(`text-gray-700`, `gap-3`)이 아니라 토큰(`text-muted-foreground`, `space-y-3`)으로 시작.

## Phase 2: 기능 안정 후 — 시스템 정리

1. 실제 쓰인 값 전수조사
2. 토큰 확정 (스페이싱 9단계 등)
3. 컴포넌트 추출 (SectionBlock 등)
4. 토큰 페이지에서 확인
5. 전체 적용

## 조립 순서

```
토큰 정의 → 컴포넌트 만들기 → 토큰 페이지에서 확인 → 화면 조립
```

이 순서를 지키면 같은 파일을 5번 건드리는 일이 없다.

**Why:** 2026-04-10 세션에서 하드코딩으로 시작 → 나중에 토큰 만들기 → 전부 교체하다가 git checkout 사고까지 남. 토큰으로 시작했으면 값만 바꾸면 끝이었음.
**How to apply:** 새 프로젝트/기능 시작 시 Phase 1 최소 토큰부터. 절대 하드코딩으로 시작하지 않기.
