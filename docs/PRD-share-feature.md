# PRD: 배너 설정 공유 기능

**작성일**: 2024-04-14
**작성자**: [PO]
**상태**: Draft

---

## 1. 목적 (WHY)

비디자이너 사용자가 만든 배너 **설정(템플릿)**을 동료에게 전달해, 동일한 설정으로 배너를 바로 사용하거나 수정할 수 있게 한다. 현재는 JSON 파일을 다운로드 후 슬랙 등으로 전달해야 하는데, 이 과정이 번거롭다.

**핵심 가치**: "이 링크(또는 코드) 하나면 내 배너 설정을 그대로 쓸 수 있다"

---

## 2. 공유 방식 비교

| 방식 | 장점 | 단점 | 백엔드 필요 | 판정 |
|------|------|------|:-----------:|:----:|
| **A. URL 해시(fragment) + gzip 압축** | 링크 하나로 즉시 공유, 서버 불필요 | 이미지 base64 포함 시 URL 매우 길어짐 (수 MB), 메신저에서 잘림 | X | **MVP 채택 (graphic 모드 전용)** |
| B. URL query parameter | A와 유사 | 서버 로그에 노출, 길이 제한 더 엄격 | X | 탈락 |
| C. 짧은 코드 (short code) | 깔끔한 URL | 코드-데이터 매핑 저장소 필요 (DB/KV) | **O** | 향후 확장 |
| D. 클립보드 JSON 복사/붙여넣기 | 구현 간단, 크기 제한 없음 | UX 불편 (복사 → 붙여넣기 2단계) | X | **MVP 보조 수단** |
| E. `.banner` 파일 다운로드 | 기존 JSON 저장과 동일 | 이미 있는 기능과 차별 없음 | X | 기존 기능 |

### MVP 결정: A + D 조합

- **주 공유 방식**: URL 해시 + gzip 압축 (graphic 모드, 이미지 미포함 상태)
- **보조 공유 방식**: 클립보드에 JSON 복사 (이미지 포함 상태에서도 동작)
- **핵심 제약**: image 모드(`imageSettings.imageDataUrl`)나 사용자 업로드 이미지(`graphicImageUrl`) 포함 시 URL 공유 불가 → 클립보드 방식으로 자동 fallback 또는 이미지 제외 옵션 제공

## 3. BannerState 직렬화 전략

### 3.1. URL 공유용 경량 상태 (ShareableState)

BannerState에서 **공유에 불필요한 필드 제거** + **이미지 데이터 분리**:

```
제외 필드:
- activeSettingsPanel (UI 상태)
- imageSettings.imageDataUrl (base64 이미지, 수십~수백 KB)
- graphicImageUrl (사용자 업로드 이미지, base64)
- blurBgUrl이 base64인 경우 (프리셋 URL은 유지)
```

### 3.2. 압축 파이프라인

```
BannerState → 불필요 필드 제거 → JSON.stringify → gzip 압축 → base64url 인코딩 → URL hash
```

예상 크기:
- graphic 모드 (이미지 없음): JSON ~2KB → gzip ~400B → base64url ~550자 → URL 길이 안전
- image 모드 (base64 이미지 포함): JSON ~수백 KB → **URL 공유 불가** → 클립보드 fallback

### 3.3. 필요 라이브러리

- `pako` (gzip 압축/해제, ~45KB gzipped) — 이미 브라우저에서 널리 사용

## 4. 사용자 플로우

### 4.1. 공유하는 쪽 (Sender)

```
1. 배너 설정을 완료한 상태에서 "공유" 버튼 클릭
2. 모달/팝오버 표시:
   a. [링크 복사] — URL 생성 후 클립보드에 복사 (graphic 모드, 이미지 미포함 시)
   b. [설정 복사] — JSON을 클립보드에 복사 (항상 가능, 이미지 포함)
3. "링크가 복사되었습니다" / "설정이 복사되었습니다" 토스트 표시
```

**조건 분기**:
- graphic 모드 + 이미지 업로드 없음 → 링크 복사 활성 + 설정 복사 활성
- graphic 모드 + 이미지 업로드 있음 → 링크 복사 시 "이미지 제외하고 공유" 안내 + 설정 복사 활성
- image 모드 → 링크 복사 비활성 (사유 표시) + 설정 복사만 활성

### 4.2. 받는 쪽 (Receiver)

**URL로 받은 경우**:
```
1. 공유 URL 클릭하여 배너 생성기 열림
2. URL 해시에서 상태 자동 디코딩 → 배너에 즉시 반영
3. "공유된 설정이 적용되었습니다" 토스트 표시
4. 자유롭게 수정 가능
```

**클립보드 JSON으로 받은 경우**:
```
1. 배너 생성기의 기존 "불러오기" 영역에 "클립보드에서 붙여넣기" 버튼 추가
2. 클릭 시 클립보드 읽기 → JSON 파싱 → 상태 적용
3. 또는 기존 JSON 파일 불러오기와 동일 UX
```

## 6. 엣지 케이스

| # | 상황 | 처리 |
|---|------|------|
| 1 | URL이 메신저에서 잘림 (2000자 초과) | 공유 시 URL 길이 체크, 초과 시 경고 + 클립보드 JSON 방식 안내 |
| 2 | 디코딩 실패 (손상된 URL) | "공유 링크가 올바르지 않습니다" 에러 토스트, 기본 상태 유지 |
| 3 | 버전 불일치 (BannerState 스키마 변경) | 공유 데이터에 `version` 필드 포함, 마이그레이션 로직 적용 (기존 `loadState`의 패치 로직 활용) |
| 4 | image 모드 설정인데 URL로 공유 시도 | 이미지 데이터가 제외됨을 명확히 안내, 수신자에게는 graphic 모드 기본 배경으로 표시 |
| 5 | 공유 URL로 진입 + 현재 작업 중인 설정 존재 | "현재 설정을 덮어씁니다. 계속하시겠습니까?" 확인 다이얼로그 |
| 6 | 클립보드 읽기 권한 미허용 | 텍스트 입력 영역(textarea) 제공하여 수동 붙여넣기 가능하게 |
| 7 | base64 이미지가 포함된 JSON 클립보드 복사 시 크기 과대 | 클립보드 API는 크기 제한 없으므로 문제 없음, 단 복사 시간이 길면 로딩 표시 |

## 7. Acceptance Criteria

### AC-1: URL 링크 공유 (P0)

- **Given** graphic 모드에서 이미지 업로드 없이 배너를 설정한 상태
- **When** "공유" > "링크 복사" 클릭
- **Then** gzip 압축된 BannerState가 URL 해시에 인코딩되어 클립보드에 복사됨
- **And** 해당 URL을 새 브라우저 탭에서 열면 동일한 배너 설정이 적용됨

### AC-2: 클립보드 JSON 공유 (P0)

- **Given** 어떤 모드/설정이든 배너를 설정한 상태
- **When** "공유" > "설정 복사" 클릭
- **Then** BannerState JSON이 클립보드에 복사됨
- **And** 다른 사용자가 "클립보드에서 붙여넣기"로 동일 설정을 불러올 수 있음

### AC-3: 공유 URL 수신 시 자동 적용 (P0)

- **Given** 공유 URL을 받은 사용자
- **When** 해당 URL로 배너 생성기에 접속
- **Then** URL 해시에서 상태가 디코딩되어 자동 적용됨
- **And** "공유된 설정이 적용되었습니다" 토스트 표시

### AC-4: 이미지 포함 시 URL 공유 제한 안내 (P1)

- **Given** 사용자 업로드 이미지가 포함된 상태
- **When** "공유" > "링크 복사" 클릭
- **Then** "이미지는 링크에 포함되지 않습니다" 안내 표시
- **And** 이미지 제외 상태로 URL 생성 or 설정 복사로 유도

### AC-5: 손상 URL 처리 (P1)

- **Given** 해시가 손상되거나 디코딩 불가능한 URL
- **When** 해당 URL로 접속
- **Then** 에러 토스트 표시, 기본 상태로 정상 로딩

### AC-6: 버전 호환성 (P1)

- **Given** 이전 버전 스키마로 생성된 공유 URL
- **When** 현재 버전 배너 생성기에서 열기
- **Then** 기존 `loadState` 마이그레이션 로직이 적용되어 정상 동작
