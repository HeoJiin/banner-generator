# PRD: 배너 관리자 — 인사이트 탭

## 1. 개요

### 배경
배너 생성기 관리자 페이지는 현재 "로그 목록" 뷰만 제공한다. 누가 언제 어떤 배너를 만들었는지는 볼 수 있지만, **데이터를 집계하여 패턴을 파악하는 기능이 없다.** 운영자가 에셋 투자 우선순위, 기능 개선 방향, 헤비유저 식별 등의 의사결정을 하려면 매번 로그를 수동으로 세어야 한다.

### 목표
- 기존 관리자 페이지에 **인사이트 탭**을 추가하여 배너 생성 데이터의 집계/시각화를 제공한다
- 운영자가 **에셋 투자, 기능 존폐, 유저 인터뷰 대상**을 데이터 기반으로 판단할 수 있게 한다
- 외부 라이브러리 없이 CSS/SVG 기반 차트로 구현한다

### 비목표 (Not Goals)
- 실시간 대시보드가 아니다 (페이지 로드 시 집계)
- 기간 필터링은 기존 년도 탭을 재활용한다 (별도 날짜 피커 없음)
- 데이터 내보내기(CSV 등)는 v1 범위 밖이다

## 2. 사용자 스토리

### US-01: 탭 전환
- **As a** 관리자, **I want** 로그 목록과 인사이트 뷰를 탭으로 전환할 수 있기를, **So that** 상황에 따라 원본 데이터와 집계 데이터를 오갈 수 있다
- 우선순위: **P0** | 복잡도: **S**
- 수용 기준:
  1. Given 관리자 페이지에 로그인한 상태, When 헤더 아래 "로그" / "인사이트" 탭이 보인다, Then 각 탭 클릭 시 해당 뷰로 전환된다
  2. Given 인사이트 탭 활성 상태, When 년도 탭을 변경한다, Then 해당 년도 데이터로 인사이트가 갱신된다

### US-02: 기능 사용률 확인
- **As a** 운영자, **I want** 그래픽형/이미지형 비율, 배너 타입별 다운로드 비율, 시즌/뱃지 사용률을 한눈에 보기를, **So that** 기능을 살릴지 죽일지 판단할 수 있다
- 우선순위: **P0** | 복잡도: **M**
- 수용 기준:
  1. Given 인사이트 탭 진입, When "기능 사용률" 섹션이 보인다, Then 모드 비율(그래픽/이미지)이 도넛 차트로 표시된다
  2. Given 배너 타입 데이터가 있을 때, When 차트를 본다, Then rolling/bm/popup/content 각각의 다운로드 횟수와 비율이 가로 막대 차트로 표시된다
  3. Given 시즌 악센트 데이터, When 차트를 본다, Then "사용 안함" vs 각 시즌별 비율이 표시된다
  4. Given 뱃지 사용 데이터, When 차트를 본다, Then 뱃지 ON/OFF 비율이 표시된다

### US-03: 키워드 선택 비율 확인
- **As a** 에셋 디자이너, **I want** 5종 키워드(benefit/info/growth/urgent/hundred) 선택 빈도를 보기를, **So that** 에셋 투자 우선순위를 결정할 수 있다
- 우선순위: **P0** | 복잡도: **S**
- 수용 기준:
  1. Given 키워드 데이터가 있을 때, When "키워드 선택 비율" 섹션을 본다, Then 5종 키워드가 빈도 내림차순으로 가로 막대 차트에 표시된다
  2. Given 각 막대, When 확인한다, Then 횟수와 전체 대비 퍼센트가 표시된다

### US-04: 하이라이트 키워드 TOP 확인
- **As a** 운영자, **I want** 타이틀에서 {중괄호}로 강조된 키워드의 빈도 랭킹을 보기를, **So that** 프리셋/자동완성 기능의 근거로 활용할 수 있다
- 우선순위: **P0** | 복잡도: **M**
- 수용 기준:
  1. Given 타이틀에 {하이라이트} 텍스트가 포함된 로그가 있을 때, When "하이라이트 키워드 TOP" 섹션을 본다, Then 상위 10개가 빈도 내림차순 리스트로 표시된다
  2. Given 각 항목, When 확인한다, Then 키워드 텍스트, 횟수, 전체 대비 비율이 보인다

### US-05: 타이틀/서브타이틀 단어 랭킹 TOP 확인
- **As a** 에셋 기획자, **I want** 배너 텍스트에서 자주 쓰이는 단어를 보기를, **So that** 새로운 에셋 키워드를 발굴할 수 있다
- 우선순위: **P1** | 복잡도: **L**
- 수용 기준:
  1. Given 타이틀+서브타이틀 텍스트 데이터가 있을 때, When "단어 랭킹 TOP" 섹션을 본다, Then 상위 15개 단어가 빈도 내림차순으로 표시된다
  2. Given 1글자 단어, 조사(은/는/이/가/을/를/의/에/로/와/과/도/만/까지/부터/에서), 불용어(그/이/저/수/것/등/및), 중괄호 기호, When 집계한다, Then 해당 단어들은 제외된다

### US-06: 인기 그래픽 에셋 TOP 확인
- **As a** 에셋 관리자, **I want** 가장 많이 사용된 그래픽 에셋 이미지를 보기를, **So that** 에셋 정리/추가 판단에 활용할 수 있다
- 우선순위: **P0** | 복잡도: **M**
- 수용 기준:
  1. Given state_snapshot에 mainGraphicUrl이 포함된 로그가 있을 때, When "인기 그래픽 에셋 TOP" 섹션을 본다, Then 상위 10개 에셋이 썸네일 + 파일명 + 횟수로 표시된다
  2. Given mode가 image인 로그, When 집계한다, Then mainGraphicUrl이 없으므로 해당 로그는 제외된다

### US-07: 사용자별 다운로드 수 확인
- **As a** 운영자, **I want** 사용자별 다운로드 횟수 랭킹을 보기를, **So that** 헤비유저 인터뷰 대상을 선정할 수 있다
- 우선순위: **P0** | 복잡도: **S**
- 수용 기준:
  1. Given 다운로드 로그가 있을 때, When "사용자별 다운로드 수" 섹션을 본다, Then 전체 사용자가 다운로드 수 내림차순으로 표시된다
  2. Given 각 항목, When 확인한다, Then 사용자 이름, 횟수, 전체 대비 비율이 보인다

## 4. API 설계

### `GET /api/insights`

기존 banner_logs 테이블을 서버사이드에서 집계하여 한 번에 반환한다.

```
Query params:
  year: string (optional) — 년도 필터, 없으면 전체

Response: {
  summary: {
    totalDownloads: number,
    uniqueUsers: number,
    graphicCount: number,
    imageCount: number,
  },

  modeRatio: {
    graphic: number,   // 건수
    image: number,
  },

  bannerTypes: {
    rolling: number,
    bm: number,
    popup: number,
    content: number,
  },

  seasonAccent: {
    none: number,
    spring: number,
    summer: number,
    autumn: number,
    winter: number,
    christmas: number,
    newyear: number,
    chuseok: number,
  },

  badgeUsage: {
    enabled: number,
    disabled: number,
  },

  keywords: {
    benefit: number,
    info: number,
    growth: number,
    urgent: number,
    hundred: number,
  },

  highlightKeywords: [
    { text: string, count: number }   // 상위 10개, 내림차순
  ],

  wordRanking: [
    { text: string, count: number }   // 상위 15개, 내림차순
  ],

  topGraphicAssets: [
    { fileName: string, url: string, count: number }  // 상위 10개
  ],

  userDownloads: [
    { userName: string, count: number }  // 전체, 내림차순
  ]
}
```

**집계 로직 요약:**

| 항목 | 데이터 소스 | 집계 방법 |
|------|-----------|----------|
| modeRatio | `mode` 컬럼 | GROUP BY mode |
| bannerTypes | `banner_types` JSON | JSON 파싱 후 각 타입 카운트 (1로그에 여러 타입 가능) |
| seasonAccent | `season_accent` 컬럼 | NULL이면 none, 그 외 값별 GROUP BY |
| badgeUsage | `state_snapshot` JSON | JSON 파싱 -> badge.enabled 확인 |
| keywords | `keywords` JSON | JSON 파싱 후 각 키워드 카운트 (1로그에 여러 키워드 가능) |
| highlightKeywords | `title` 컬럼 | 정규식 `\{([^}]+)\}` 로 추출, 빈도 집계 |
| wordRanking | `title` + `subtitle` | 형태소 분리(공백+특수문자 기준), 불용어 제거, 빈도 집계 |
| topGraphicAssets | `state_snapshot` JSON | mainGraphicUrl에서 파일명 추출, 빈도 집계 |
| userDownloads | `user_name` 컬럼 | GROUP BY user_name, COUNT |

## 10. 디자이너 관점 리뷰 코멘트

### 전반적 구조
1. **정보 과부하 주의** — 6개 인사이트를 한 페이지에 세로로 나열하면 스크롤이 매우 길어진다. **기능 사용률 섹션 내 4개 차트를 2x2 그리드로 배치**하고, 이후 섹션들도 가능한 곳은 2단 레이아웃을 사용하여 밀도를 높인다.
2. **시각적 위계** — 요약 카드(Summary Cards)가 가장 먼저 눈에 들어와야 한다. 섹션 제목은 `text-h3`으로 통일하되, 요약 카드의 숫자만 `text-display`로 차별화한다.

### 차트 관련
3. **도넛 차트 접근성** — 색상만으로 구분하지 말고, 각 세그먼트 옆에 반드시 텍스트 범례를 배치한다. 색맹 사용자를 위해 패턴(줄무늬 등)까지는 v1에서 불필요하지만, 범례는 필수다.
4. **가로 막대 차트 — 0% 처리** — 값이 0인 항목도 행 자체는 표시하되, 막대가 없고 값만 "0 (0%)"으로 표기한다. 행을 아예 숨기면 "이 기능은 있는데 아무도 안 쓴다"는 인사이트 자체가 사라진다.
5. **막대 색상 단조로움** — 모든 막대가 `var(--accent)` 하나면 시각적으로 밋밋하다. 1위 항목만 `var(--accent)`, 나머지는 `var(--accent)` 에 `opacity: 0.6`을 적용하거나, 배너 타입별로 고유 색상을 쓰는 것을 고려한다. 다만 v1에서는 단색 + opacity 그라데이션으로 충분하다.

### 레이아웃 관련
6. **에셋 그리드 5열은 960px에서 빡빡하다** — `max-width: 960px` 기준으로 5열이면 카드당 약 176px. 이미지 + 텍스트를 담기에 최소한이다. **4열(약 220px/카드)을 기본으로 하고**, 필요 시 5열은 더 넓은 뷰포트에서만 적용한다.
7. **단어 랭킹 2단** — 15개를 2단으로 나누면 좌: 8개, 우: 7개. 시각적으로 약간 비대칭이지만 허용 범위다. 다만 숫자가 적을 때(5개 이하) 2단이 어색해지므로, **8개 이하일 때는 1단으로 fallback** 하는 조건을 추가한다.

### 인터랙션 관련
8. **호버 피드백** — 가로 막대에 호버 시 해당 행만 배경색 `var(--fill-tertiary)` 로 하이라이트하면 데이터를 읽기 쉬워진다.
9. **툴팁 불필요** — 값이 이미 행에 표시되므로 별도 툴팁은 과잉. 구현 복잡도 대비 가치가 낮다.

### 데이터 빈 상태
10. **부분 빈 상태 처리** — 전체 데이터가 없는 것과, 특정 섹션만 비어있는 것(예: 아무도 시즌 악센트를 안 쓴 경우)은 다르게 처리해야 한다. 섹션 내 데이터가 0일 때 차트 대신 "데이터 없음" 텍스트를 표시한다.
