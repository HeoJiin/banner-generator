## 1. 배경 및 문제 정의

### 현재 상황
카카오스타일 파트너센터에서 사용하는 광고/톡채널/기사 이미지용 배너는 **디자이너가 피그마에서 직접 제작**하고 있다.

### 문제
| 문제 | 영향 |
|------|------|
| 배너 제작마다 디자이너 리소스 소요 | 디자이너 병목, 배너 요청 대기 시간 증가 |
| 비디자이너는 배너를 직접 만들 수 없음 | 마케터/PM의 자율성 저하 |
| 피그마 가이드가 있지만 활용 진입장벽 높음 | 가이드 존재에도 셀프서비스 불가 |

### WHY
디자이너 리소스를 반복적인 배너 제작에서 해방하고, 마케터/운영팀이 가이드 내에서 셀프서비스로 배너를 생성할 수 있도록 한다.

---

## 2. 제품 비전

> **피그마 가이드를 그대로 웹 UI로 옮겨서, 누구나 클릭 몇 번으로 파트너센터 배너 한벌을 만든다.**

### 성공 지표 (KPI)
| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 디자이너 배너 제작 요청 감소율 | 70% 이상 감소 | 디자이너 작업 요청 건수 비교 |
| 비디자이너 배너 생성 건수 | 월 50건 이상 | 다운로드 이벤트 추적 |
| 배너 제작 소요 시간 | 5분 이내 | 사용자 세션 시간 |

## 3. 대상 사용자

### Primary: 파트너센터 운영/마케팅 담당자
- 피그마 사용 경험 없음
- 배너 시안을 빠르게 만들어 적용해야 함
- 디자인 감각보다는 가이드 내에서 선택하는 방식 선호

### Secondary: 디자이너
- 빠른 시안 확인용으로 사용
- 가이드 기반 배너를 직접 만드는 대신 툴로 위임

## 4. 배너 모드: 그래픽형 vs 이미지형

배너는 비주얼 소스에 따라 **두 가지 모드**로 나뉜다.

### 그래픽형 (기본)
- **비주얼**: 소구점 키워드별 3D 그래픽 에셋 + 시즌 강조 조합
- **특징**: 배경색/그라데이션 + 그래픽 오브제 + 배경 꾸밈 + 텍스트 조합
- **대상 배너**: 4종 모두 (롤링, BM, 팝업, 컨텐츠)

### 이미지형
- **비주얼**: 사용자가 업로드한 사진/이미지를 메인 비주얼로 사용
- **특징**: 업로드 이미지 위에 그라데이션 오버레이 + 텍스트
- **이미지 배치**:
  - 롤링: 우측 정렬, width 400px, 좌→우 검은색 그라데이션
  - BM: 우측 정렬, width 200px, 좌→우 검은색 그라데이션
  - 팝업: 가로 꽉 채움 하단 정렬, 위→아래 그라데이션
  - 컨텐츠: 가로 꽉 채움, 위→아래 그라데이션
- **그라데이션 설정** (팝업/컨텐츠):
  - 프리셋: 블랙 (기본) / 화이트
  - 커스텀: HEX 직접 지정
  - 투명도 슬라이더: 40~80% 조절
- **대상 배너**: 4종 모두

## 5. 소구점 키워드 + 시즌 강조 시스템

### 5-1. 소구점 키워드 (필수 선택, 1개)

| ID | 라벨 | 테마 색상 계열 | 그래픽 성격 |
|---|---|---|---|
| benefit | 혜택 | 블루/퍼플/레드/골드 | 선물, 돈, 트럭, 복주머니 |
| info | 정보 제공 | 블루/시안 | 돋보기, 연필, 책 |
| growth | 성장 | 블루/퍼플/시안/그린 | 그래프, 깃발 |
| urgent | 긴급 | 블랙/다크레드/다크퍼플 | 폭탄, 불시계, 번개 |
| hundred | 100원 | 레드/블랙/골드 | 100원 전용 에셋 |

키워드 선택 시 자동 설정:
- 배경색/그라데이션 (테마 풀에서 랜덤)
- 메인 그래픽 에셋 (키워드 풀에서 랜덤)
- 블러 배경 에셋
- 배경 꾸밈 (컨페티 등)

### 5-2. 시즌 강조 (선택사항)

| ID | 라벨 |
|---|---|
| spring | 봄 |
| summer | 여름 |
| autumn | 가을 |
| winter | 겨울 |
| christmas | 크리스마스 |
| newyear | 신년 |
| chuseok | 추석 |

시즌 강조 선택 시 추가 변경:
- 배경색 → 시즌 테마로 교체
- 블러 배경 → 시즌 전용 에셋
- 배경 꾸밈 → 시즌 전용 (꽃, 눈, 나무, 모래성 등)
- **그래픽 에셋 → 키워드+시즌 조합 풀 우선 (비중 75%), 범용 키워드 에셋 (비중 25%)**

### 5-3. 동적 에셋 로딩 시스템

파일명 기반 자동 매핑 — 코드 수정 없이 파일 추가만으로 확장 가능.

**API**: `GET /api/assets` → `public/assets/3d/` 스캔

**그룹핑 규칙**:
| 파일명 | 그룹 | 설명 |
|---|---|---|
| `혜택 선물과 돈1-1.png` | `benefit` | 범용 혜택 (시즌 무관) |
| `혜택 크리스마스 1.png` | `benefit_christmas` | 혜택+크리스마스 전용 |
| `성장 신년 1.png` | `growth_newyear` | 성장+신년 전용 |

**시즌 분리 원칙**:
- 시즌 토큰이 포함된 파일 → 조합 그룹에만 등록 (기본 키워드 그룹에 빠짐)
- 시즌 없이 키워드만 선택 시 → 시즌 전용 에셋이 나오지 않음
- 시즌 강조 선택 시 → 조합 에셋 75% + 범용 에셋 25% 비중 혼합

**키워드 토큰**: 혜택, 성장, 긴급, 100원, 정보 제공, 정보
**시즌 토큰**: 봄, 여름, 가을, 겨울, 크리스마스, 신년, 추석

### 5-4. 다시 뽑기

| 버튼 | 동작 |
|---|---|
| 🎲 소구점 다시 뽑기 | 배경 + 그래픽 + 블러 + 꾸밈 전체 랜덤 재선택 |
| 🎲 시즌 배경 다시 뽑기 | 배경 + 블러 + 꾸밈 + 그래픽 시즌 기준 재선택 |

## 6. 배경 시스템

### 6-1. 배경색
- **단색**: 프리셋 컬러 팔레트 + HEX 직접 입력
- **그라데이션**: 프리셋 8종 (시작색→끝색 2색) + 세부 색상 피커 + 방향 회전 (0~315° 8방향)

### 6-2. 배경 꾸밈 (BgDecoration)

| 카테고리 | 타입 |
|---|---|
| 겨울 | winter, winter2 |
| 꽃 | flower1, flower2, flower3 |
| 컨페티 | confetti, confetti2, confetti3 |
| 여름 | tree (나무), sandcastle (모래성) |

- 배너별 전용 PNG (팝업/롤링/컨텐츠/BM 각각 별도 파일)
- 투명도 슬라이더 (20~100%)
- 시즌 강조 선택 시 자동 적용

### 6-3. 블러 배경
- 독립 토글
- 메인 그래픽 또는 시즌 블러 에셋을 블러(25~30px) 처리하여 배경에 깔기

## 10. 다운로드

- **전체 다운로드**: 활성화된 배너 4종 → 2x PNG → ZIP
- **개별 다운로드**: 배너별 2x PNG
- **ZIP 파일명**: `banners_{YYYYMMDD}_{HHmmss}.zip`
- **내부 파일명**: `{배너타입}_{W}x{H}@2x.png`
