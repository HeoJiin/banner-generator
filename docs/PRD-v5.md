## 1. 배경 및 문제 정의

### 1-1. 현재 상황 (As-Is)

카카오스타일 파트너센터에서 사용하는 광고 배너(상단 롤링, BM, 팝업, 컨텐츠)는 **디자이너가 Figma에서 직접 제작**하는 프로세스로 운영되고 있었습니다. 배너 요청은 파트너팀, 마케팅팀, 운영팀 등 다양한 부서에서 발생하며, 시즌 프로모션 시기에는 요청량이 급증합니다.

### 1-2. 핵심 문제

| # | 문제 | 영향 | 심각도 |
|---|------|------|--------|
| 1 | 배너 제작마다 디자이너 리소스 소요 | 디자이너 병목, 핵심 디자인 업무 지연 | **Critical** |
| 2 | 비디자이너는 배너를 직접 만들 수 없음 | 마케터/운영자의 자율성 저하, 배너 요청 대기 시간 증가 | **High** |
| 3 | Figma 가이드가 있지만 활용 진입장벽 높음 | 가이드 존재에도 셀프서비스 불가 | **High** |
| 4 | 배너 품질 편차 | 제작자에 따라 브랜드 가이드라인 준수 정도 상이 | **Medium** |
| 5 | 시즌/캠페인 대응 속도 부족 | 프로모션 일정에 배너 준비가 지연 | **High** |

### 1-3. WHY — 해결 방향

> 디자이너 리소스를 반복적인 배너 제작에서 해방하고, **비디자이너가 브랜드 가이드라인 내에서 셀프서비스로 배너를 생성**할 수 있는 웹 기반 도구를 제공합니다.

## 5. 기능 명세

### 5-1. 배너 타입 시스템

배너 생성기는 파트너센터에서 사용하는 **4종 배너**를 동시에 생성합니다.

| 배너 타입 | 사이즈 (px) | 용도 |
|-----------|-------------|------|
| 상단 롤링 배너 | 1330 x 128 | 파트너센터 상단 띠배너 |
| BM 배너 | 665 x 146 | 파트너센터 BM 영역 |
| 이미지 팝업 | 520 x 520 | 파트너센터 진입 시 정사각형 팝업 |
| 컨텐츠 팝업 | 1000 x 600 | 도메인 진입 시 가로형 팝업 |

**구현 상태**: 4종 모두 완료. 각 배너는 독립 템플릿 컴포넌트로 구현 (`RollingBanner`, `BmBanner`, `PopupBanner`, `ContentBanner`).

### 5-2. 듀얼 모드: 그래픽형 / 이미지형

| 모드 | 비주얼 소스 | 특징 |
|------|-------------|------|
| **그래픽형** (기본) | 3D 오브제 에셋 + 배경색/그라데이션 | 소구점 키워드 → 자동 테마 세팅 |
| **이미지형** | 사용자 업로드 사진 | 이미지 + 그라데이션 오버레이 + 텍스트 |

**이미지형 상세**:
- 배너별 이미지 배치 규칙: 롤링(우측 400px), BM(우측 200px), 팝업(하단 꽉 채움), 컨텐츠(가로 꽉 채움)
- 그라데이션 프리셋: 블랙/화이트/커스텀 HEX
- 투명도 슬라이더: 40~100%
- 배너별 이미지 영역 미세 조절: 그라데이션 범위, 이미지 크기 배율(0.5~1.5x), 좌우/상하 이동(-200~200px)
- 이미지 업로드 시 자동 색상 추출 → 배경색/텍스트 색상 자동 대비 설정

**구현 상태**: 완료. `ImageOverlay.tsx`에서 배너 타입별 그라데이션/이미지 영역 제어 구현.

### 5-3. 소구점 키워드 시스템 (5종)

| 키워드 ID | 라벨 | 테마 색상 계열 | 그래픽 성격 |
|-----------|------|----------------|-------------|
| `benefit` | 혜택 | 블루/퍼플/레드/골드 | 선물, 돈, 트럭, 복주머니 |
| `info` | 정보 제공 | 블루/시안 | 돋보기, 연필, 책 |
| `growth` | 성장 | 블루/퍼플/시안/그린 | 그래프, 깃발 |
| `urgent` | 긴급 | 블랙/다크레드/다크퍼플 | 폭탄, 불시계, 번개 |
| `hundred` | 100원 | 레드/블랙/골드 | 100원 전용 에셋 |

**키워드 선택 시 자동 설정**:
1. 배경색/그라데이션 — 키워드별 50+ 컬러 베리에이션 풀에서 가중치 기반 랜덤 선택
2. 메인 그래픽 에셋 — 키워드 풀에서 랜덤 (2개: 메인 + 서브)
3. 블러 배경 에셋
4. 배경 꾸밈 (컨페티 등)

**구현 상태**: 완료. 키워드별 배경 팔레트에 비비드/파스텔/다크/그라데이션 계열 총 60+ 베리에이션 구현.

### 5-4. 시즌 강조 시스템 (7종)

| 시즌 ID | 라벨 | 전용 컬러 베리에이션 |
|---------|------|---------------------|
| `spring` | 봄 | 25+ |
| `summer` | 여름 | 25+ |
| `autumn` | 가을 | 25+ |
| `winter` | 겨울 | 25+ |
| `christmas` | 크리스마스 | 25+ |
| `newyear` | 신년 | 25+ |
| `chuseok` | 추석 | 25+ |

**시즌 선택 시 변경 사항**:
- 배경색 → 시즌 전용 컬러 팔레트로 교체
- 블러 배경 → 시즌 전용 에셋
- 배경 꾸밈 → 시즌 전용 (꽃, 눈, 나무, 모래성, 단풍 등)
- 그래픽 에셋 → 키워드+시즌 조합 풀 우선 (75%) + 범용 키워드 에셋 (25%)

**구현 상태**: 완료. 7개 시즌 모두 전용 컬러 팔레트 및 에셋 매핑 구현.

### 5-5. 동적 에셋 로딩 시스템

| 항목 | 내용 |
|------|------|
| **3D 그래픽 에셋** | 881개 PNG |
| **배경 꾸밈 에셋** | 100개 PNG |
| **API** | `GET /api/assets` → `public/assets/3d/` 디렉토리 스캔 |
| **그룹핑** | 영어 접두사 기반 자동 분류 (benefit, info, growth, urgent, hundred + 시즌 조합) |

**핵심 설계 원칙**: 코드 수정 없이 파일 추가만으로 에셋 확장 가능.

**파일명 기반 자동 매핑 규칙**:
- `benefit-gift-01.png` → `benefit` 그룹 (범용 혜택)
- `benefit-christmas-01.png` → `benefit_christmas` 그룹 (혜택+크리스마스 전용)
- 시즌 토큰이 포함된 파일은 조합 그룹에만 등록 (기본 키워드 그룹에서 제외)

**구현 상태**: 완료. 영어 파일명 체계 전환 완료, 동적 로딩 API 구현.

### 5-6. 배경 시스템

#### 배경색
- **단색**: 프리셋 16색 + HEX 직접 입력
- **그라데이션**: 프리셋 8종(2색) + 세부 색상 피커 + 방향 회전(0~315도, 8방향)
- 색상 변경 시 텍스트 색상 자동 대비 (밝은 배경 → 어두운 텍스트, 반대도 동일)

#### 배경 꾸밈 (BgDecoration)

| 카테고리 | 베리에이션 수 | 종류 |
|----------|-------------|------|
| 겨울 | 4 | winter 1~4 |
| 꽃 | 3 | flower 1~3 |
| 여름 | 4 | 나무, 모래성, 튜브, 파도 |
| 가을 | 3 | 낙엽, 단풍, 열매 |
| 컨페티 | 4 | confetti 1~4 |

- 배너별 전용 PNG (팝업/롤링/컨텐츠/BM 각각 별도)
- 투명도 슬라이더 (20~100%)
- 시즌 강조 선택 시 자동 적용

#### 블러 배경
- 독립 토글
- 메인 그래픽 또는 시즌 블러 에셋을 블러(25~30px) 처리하여 배경에 적용

**구현 상태**: 완료. 5개 카테고리 18개 베리에이션 구현.

## 11. 기술 아키텍처

| 항목 | 결정 |
|------|------|
| 프레임워크 | Next.js 14 + TypeScript |
| 스타일링 | Tailwind CSS + 시맨틱 컬러 토큰 |

| 에셋 로딩 | 동적 — API `/api/assets`가 `public/assets/3d/` 스캔, 영어 접두사 기반 자동 분류 |
| 상태 관리 | Custom Hook (useBannerStore) — React useState 기반 |
| 데이터 저장 | localStorage (최대 30 슬롯, 버전 마이그레이션 포함) |
| 색상 추출 | Canvas API 기반 이미지 색상 추출 (자동 배경/텍스트 대비) |
