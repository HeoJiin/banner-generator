# PRD: 배너 생성기 관리자 모드 (Admin Mode)

## 1. 개요

### 배경
배너 생성기는 현재 순수 클라이언트사이드 앱으로, 누가 어떤 배너를 만들었는지 추적할 방법이 없다. 운영 관점에서 배너 생성 현황(누가, 언제, 어떤 내용으로)을 파악할 필요가 있다.

### 목표
- 배너를 다운로드한 사용자와 배너 내용(텍스트 + 이미지)을 기록
- 관리자가 한눈에 조회할 수 있는 대시보드 제공

### 제약 조건
- **기존 배포(정적 사이트)는 건드리지 않는다** — 현재 사용자에게 영향 없음
- Railway에 별도 API 서버를 배포하여 로깅 처리
- 배너 생성기에는 최소한의 코드만 추가 (다운로드 시 API 호출)

```
┌─────────────────────┐         ┌──────────────────────────┐
│  배너 생성기 (기존)    │  POST   │  banner-admin-api (신규)  │
│  정적 사이트           │ ──────→ │  Railway 별도 서비스       │
│                       │         │                          │
│  다운로드 클릭 시       │         │  Express + SQLite        │

│                       │         │  ├ GET  /api/logs/:id    │
│                       │         │  └ /admin (관리자 UI)     │
└─────────────────────┘         └──────────────────────────┘
```

## 3. 데이터 모델

### BannerLog (로그 1건 = 다운로드 1회)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | TEXT (UUID) | PK |
| created_at | DATETIME | 다운로드 시각 |
| user_name | TEXT | 다운로드한 사람 이름 (최초 1회 입력, 이후 캐싱) |
| title | TEXT | 배너 타이틀 텍스트 |
| subtitle | TEXT | 배너 서브타이틀 텍스트 |
| keywords | TEXT (JSON) | 선택된 키워드 배열 ["benefit", "urgent"] |
| season_accent | TEXT | 시즌 악센트 (nullable) |
| mode | TEXT | 'graphic' \| 'image' |
| background_color | TEXT | 배경색 hex |
| instances_count | INTEGER | 다운로드된 배너 인스턴스 수 |
| banner_types | TEXT (JSON) | 포함된 배너 타입 ["rolling", "popup"] |
| state_snapshot | TEXT (JSON) | BannerState 전체 스냅샷 (상세 조회용) |

### BannerThumbnail (로그 1건에 N개 이미지)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | TEXT (UUID) | PK |
| log_id | TEXT | FK → BannerLog.id |
| banner_type | TEXT | 'rolling' \| 'bm' \| 'popup' \| 'content' |
| instance_label | TEXT | "상단 롤링 배너", "BM 배너 (2)" 등 |
| image_data | BLOB | PNG 썸네일 (축소 버전, 원본 대비 ~25%) |
| width | INTEGER | 원본 너비 |
| height | INTEGER | 원본 높이 |

## 4. API 명세

### `POST /api/logs` — 로그 저장
```
Content-Type: multipart/form-data

Fields:
  user_name: string
  title: string
  subtitle: string
  keywords: string (JSON array)
  season_accent: string
  mode: string
  background_color: string
  instances_count: number
  banner_types: string (JSON array)
  state_snapshot: string (JSON)
  
Files:
  thumbnails[0]: PNG file  (field name: thumbnail_rolling)
  thumbnails[1]: PNG file  (field name: thumbnail_bm)
  ...
```

Response: `201 { id, created_at }`

### `GET /api/logs` — 로그 목록 조회
```
Query params:
  page: number (default 1)
  limit: number (default 20)
  search: string (이름 또는 타이틀 검색)
  
Response: {
  logs: BannerLog[],
  total: number,
  page: number,
  totalPages: number
}
```

### `GET /api/logs/:id` — 로그 상세 + 썸네일
```
Response: {
  log: BannerLog,
  thumbnails: { banner_type, instance_label, image_url, width, height }[]
}
```

### `GET /api/thumbnails/:id` — 썸네일 이미지 바이너리
```
Response: image/png
```

## 5. 사용자 플로우

### 5-1. 배너 생성기 사용자 (기존 + 변경)

```
[배너 편집] → [전체 다운로드 ZIP 클릭]
                    │
                    ▼
          ┌─ 이름 저장됨? ──┐
          │ Yes             │ No
          ▼                 ▼
     바로 다운로드    이름 입력 팝업
          │              │ 입력 후 확인
          ▼              ▼
     백그라운드로 API 전송 (다운로드 차단 없음)
          │
          ▼
     ZIP 다운로드 완료
```

- **이름 입력은 최초 1회만** — localStorage에 캐싱
- **API 전송 실패해도 다운로드는 정상 진행** (fire-and-forget)
- 사용자 경험에 영향 없음

### 5-2. 관리자 (신규)

```
[/admin 접속] → [로그 목록 화면]
                     │
                     ├─ 검색 (이름/타이틀)
                     ├─ 페이지네이션
                     │
                     ▼
              [카드 클릭 → 상세 보기]
                     │
                     ├─ 배너 설정 정보 확인
                     ├─ 썸네일 이미지 프리뷰
                     └─ 원본 크기로 확대 보기
```

- **기존 기능에 영향 없음** — API 실패해도 다운로드 정상 작동

## 8. 기술 스택 (admin-api 서버)

| 항목 | 선택 |
|------|------|
| Runtime | Node.js |
| Framework | Express |
| DB | SQLite (better-sqlite3) — Railway volume에 저장 |
| 이미지 저장 | SQLite BLOB (썸네일 소용량이므로 파일시스템 불필요) |
| 빌드 | TypeScript + esbuild |
| 배포 | Railway (별도 서비스) |

---

## 9. 비기능 요구사항

- 썸네일은 원본 대비 25% 크기로 리사이즈하여 저장 (용량 절감)
- API 호출은 fire-and-forget (다운로드 UX 차단 없음)
- CORS: 배너 생성기 도메인만 허용
- 관리자 페이지 접근 제한: 간단한 비밀번호 (환경변수)
- SQLite 파일은 Railway Volume에 마운트하여 영속성 확보
